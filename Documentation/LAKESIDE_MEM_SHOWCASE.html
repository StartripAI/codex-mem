Lakeside-mem | Developer Memory Runtime

Token is the hard currency of modern work and research.

Lakeside-mem helps teams stop re-paying context costs every session. It runs tasks through a 6-layer memory runtime, retrieves the minimum useful context, and returns evidence-backed output plus the next executable command.

Core Workflow

  1. Task intake via `run-target` / `run-target-auto`
  2. Plan compile into `TaskSpec` + `ExecutionPlan`
  3. Memory writes into SQLite + FTS + graph-lite
  4. Hybrid retrieval (lexical + structure + graph-lite + optional embeddings)
  5. Single-model execution (`none|codex|claude`)
  6. Coverage gate + critic + forced next command delivery

Why Teams Like It

  1. Less repeated context dumping
  2. Faster onboarding in unfamiliar repos
  3. More stable long-running task output
  4. Stronger traceability with file/symbol evidence
  5. Practical CLI-first workflow, no store dependency

Most-Used Scenarios (How To Use)

1) First-day onboarding

Learn architecture, entrypoint, persistence, AI path, tests, and risks with evidence.

bash Scripts/lakeside_mem.sh run-target "/ABS/PATH/TO/TARGET_PROJECT" --project target \
  --question "learn this project: goal, architecture, module map, entrypoint, main flow, persistence, ai generation, tests, risks"

2) Bug triage and root cause

Turn an issue into repro path, root cause chain, and minimal-risk fix plan.

bash Scripts/lakeside_mem.sh run-target "/ABS/PATH/TO/TARGET_PROJECT" --project target \
  --question "triage this bug: repro path, root cause chain, minimal-risk fix, validation checklist"

3) Debug failing tests / terminal errors

Isolate failure points quickly and return the smallest reliable patch direction.

bash Scripts/lakeside_mem.sh run-target "/ABS/PATH/TO/TARGET_PROJECT" --project target \
  --question "debug this failure: isolate failing path, evidence, smallest reliable fix"

4) Feature implementation

Implement with boundaries, compatibility checks, and validation steps.

bash Scripts/lakeside_mem.sh run-target "/ABS/PATH/TO/TARGET_PROJECT" --project target \
  --question "implement this feature with minimal patch, compatibility boundaries, tests, rollout notes"

5) Large-codebase refactor

Map dependencies and execute staged refactors with regression guardrails.

bash Scripts/lakeside_mem.sh run-target "/ABS/PATH/TO/TARGET_PROJECT" --project target \
  --question "plan a safe refactor: dependency map, blast radius, staged steps, regression checks"

6) PR review / incident forensics / handoff

Use the same entrypoint for review risk scans, timeline reconstruction, and clean team handoffs.

bash Scripts/lakeside_mem.sh run-target "/ABS/PATH/TO/TARGET_PROJECT" --project target \
  --question "review this change or incident timeline, list evidence, open risks, and next executable command"
Lakeside-mem is a straightforward tool for a simple problem: repeated context is expensive. Keep memory persistent, pull only what you need, and keep moving.
